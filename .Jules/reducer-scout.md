## 2025-05-23 - Cart Screen State Refactor **Learning:** In `CartScreenComponent`, the modal state (`editingItem`) and async loading state (`updatingItemId`) were entangled. The modal was closed immediately on save, rendering the `isLoading` prop in the modal useless. Refactoring to a reducer highlighted this interaction, though preserving exact behavior meant keeping the immediate close. **Action:** Future refactors involving modals and async actions should carefully check if the modal is intended to stay open during loading, and if so, the reducer state should reflect a "submitting" state that keeps the modal open.
