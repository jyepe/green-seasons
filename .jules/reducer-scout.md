## 2024-05-23 - State Mirroring in Favorites **Learning:** I found `stepperItems` local state being synced with `cartItems` server data via `useEffect`. This "state mirroring" causes redundant renders and potential de-syncs. **Action:** When server data (cart) drives local UI state (steppers), either derive the value directly during render or, if local overrides are needed (optimistic updates), use a reducer to handle "SYNC" actions atomically rather than reacting with effects.
